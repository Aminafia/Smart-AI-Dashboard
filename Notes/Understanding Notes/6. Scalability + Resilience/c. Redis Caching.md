# Redis Caching Understanding

# Why Caching Exists

Caching improves:
- performance
- scalability
- response times

Reduces:
- repeated DB queries
- repeated API calls
- expensive computations

---

# Redis

Redis is in-memory distributed cache.

Very fast because:
- data stored in RAM

---

# Distributed Cache

Shared across multiple servers.

Important for:
- scalable cloud systems
- load-balanced systems
- microservices

---

# Cache Use Cases

Examples:
- AI responses
- session data
- frequently requested data
- rate limiting counters

---

# Important Understanding

Caching trades:
- memory usage
FOR
- performance improvement

- large-scale APIs
